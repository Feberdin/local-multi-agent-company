Web Search Providers

SearXNG zuerst, Brave optional als Fallback. Trusted Sources behalten immer Vorrang.

Wichtig fuer Unraid

Dieser Stack startet standardmaessig keinen eigenen SearXNG-Container. Trage hier daher eine bestehende SearXNG-Instanz ein oder lasse den Provider deaktiviert.

Ein typischer JSON-Endpunkt ist /search. Wenn ein 404 erscheint, stimmen meist base_url oder search_path nicht mit der echten SearXNG-Instanz ueberein.

Fuer die Worker zaehlt nur die JSON-API. Browser-HTML allein reicht nicht aus. Wenn der JSON-Healthcheck scheitert, pruefe in der SearXNG-settings.yml besonders:

search:
  formats:
    - html
    - json

Brave ist in diesem Stack nur ein optionaler Fallback. Laut der oeffentlichen Brave-Preisuebersicht vom 11. April 2026 solltest du fuer neue Setups von kostenpflichtigen Search-/Answers-Tarifen mit monatlichem Guthaben ausgehen, nicht von einem dauerhaft freien Default.
